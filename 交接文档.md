## 发布与验收边界

- 本地首个提交为 `3e42e9d feat: add bilingual CLI support`；公开 GitHub 仓库已创建：`https://github.com/1yang2471/localenv-guard`。安全改动已部署；由于命令行 HTTPS/API 网络仍不可用，部署通过 GitHub 网页完成。
- 首次远程运行 `CI #1`（提交 `3280f45`）的 6 个 macOS/Linux 任务失败，原因是 Windows 专属 `npm.cmd` 单测在非 Windows 平台仍强制执行，并非产品运行故障。已改为平台条件测试，并新增非 Windows 的 npm 原命令断言。
- 第二次远程运行 `CI #2`（提交 `664a858`）只在 macOS Node 18 失败：不同进程型集成测试并发运行，夹具互相占用端口，产品按三次避让上限停止。已将 npm 测试命令固定为 `--test-concurrency=1`；修复后的提交尚待远程矩阵验证。
- 先前 GitHub CLI 的网页授权可用，但命令行 HTTPS/API 网络问题阻断了授权流程；本次未重新验证该外部状态。
- 真实 Vite 手工 E2E 曾多次受到残留进程和操作顺序干扰，不能作为验收证据。当前有效的 Windows 依据应以自动化测试为准；测试不全绿时也不构成有效发布证据。
- GitHub Actions `CI #5`（Run ID `29914452784`，提交 `55afdc7`）状态为 `Success`，明确显示 `9 jobs completed`，覆盖 Windows、macOS、Linux 与 Node.js 18、20、22。该提交满足当时部署验收门槛；未执行 npm 发布或创建 Git tag。
- `CI #6` 曾因网页上传误把 `cli-args.test.js`、`protected-services.test.js`、`runner.js`、`protected-services.js` 和安全报告放到根目录而失败。这些重复文件均已删除；根目录的 `交接文档.md` 属于正式文件并保留。
- GitHub Actions `CI #11`（Run ID `29918455018`，提交 `eaa3c35`）状态为 `Success`，明确显示 `9 jobs completed`，是当前安全改动的发布验收证据；未执行 npm 发布或创建 Git tag。
