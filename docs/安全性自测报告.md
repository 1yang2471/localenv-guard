# LocalEnv-Guard 安全性自测报告

日期：2026-07-22

## 红线 1：核心服务熔断

实现位置：`src/protected-services.js`

保护名单覆盖：

- MySQL / mysqld
- PostgreSQL / postgres / postmaster
- Redis
- MongoDB
- Nginx
- Apache / httpd
- Docker / dockerd / Docker Desktop
- SSH / sshd / ssh-agent
- Elasticsearch
- RabbitMQ
- Kafka / Zookeeper
- root、SYSTEM、LOCAL SERVICE、NETWORK SERVICE 等高权限账户归属进程
- Windows 核心服务宿主（如 svchost.exe、services.exe、lsass.exe）
- Unix 初始化服务（如 systemd、launchd）
- 未能明确识别为本地开发服务器的未知进程，默认不提供终止选项
- MariaDB、Microsoft SQL Server、Oracle、CockroachDB、Cassandra、Memcached、CouchDB、Neo4j、InfluxDB、ClickHouse、OpenSearch、Solr、NATS、Pulsar、etcd、Consul、Vault、MinIO、Kubernetes、IIS、Caddy、Traefik、HAProxy、Envoy 和 Windows 安全服务等

自测证据：

- `test/protected-services.test.js` 验证 PostgreSQL、Docker、高权限账户和 Windows 核心服务宿主会被标记为保护服务。
- `test/doctor.test.js` 验证 `doctor` 对保护服务的建议动作是“手动处理”，不是“可交互清理”。
- `test/protected-services.test.js` 与 `test/doctor.test.js` 验证未知服务不会获得终止选项，只有明确的开发命令可以交互清理。
- 保护服务与开发框架回归集覆盖 82 个命令案例，验证受保护服务优先于开发关键字匹配。

## 红线 1.1：确认后的监听者复核

实现位置：`src/protected-services.js`、`src/runner.js`、`src/doctor.js`

- 在用户选择终止后，重新查询该端口的监听者。
- 只有同一 PID 仍在监听、完整命令行仍一致、可获得的启动时间仍一致，且进程仍是明确开发服务时，才会终止其进程树。
- PID 不存在、命令变化、可获得的启动时间变化，或占用者变成保护服务、未知进程时，都会停止终止。
- 该复核同时覆盖常规冲突恢复和 `leg doctor` 的逐项清理。
- `--auto` 不参与终止决策：无论占用者类型如何，自动模式只会选择可用端口重试；保护服务仍会中断。
- Go/Rust 通用运行器以及 Streamlit、Gradio、Rails、Phoenix、Spring Boot 和模糊 `air` 命令缺少足够的开发环境证据，不提供终止选项。

## 红线 2：权限优雅降级

实现位置：`src/killer.js`

当终止进程时遇到 `Access denied`、`Access is denied`、`Operation not permitted`、`EPERM`、`拒绝访问` 等错误，会转换为中文提示：

```text
当前权限不足，未能终止目标进程。请确认该进程是否属于管理员/root，必要时用 sudo 或管理员终端手动处理。
```

底层异常不会直接暴露给普通用户。

## 红线 3：进程树彻底清理

实现位置：`src/killer.js`

- Windows：优先调用 `taskkill.exe /PID <pid> /T /F` 递归终止进程树；若受限环境拒绝该命令，回退到 Windows 原生进程快照递归查找并终止子进程。权限仍不足时显示中文提示，不抛出底层堆栈。
- macOS/Linux：通过 `pgrep -P <pid>` 递归查找子进程，先发送 `SIGTERM`，短暂等待后只对仍存活的进程发送 `SIGKILL`。

## 红线 4：跨平台一致性

实现位置：`src/process-info.js`、`src/killer.js`

- Windows 优先使用 PowerShell 原生命令获取监听端口和进程信息；若环境不返回监听项，则回退到 `netstat -ano` 解析 PID，再用 `Get-Process` 补充进程名和路径。
- macOS/Linux 使用 `lsof` 与 `ps` 获取监听端口和进程信息。
- CLI 交互和决策选项在三个平台保持一致。
